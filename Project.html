Software Project (Due, September 22th)

[DLX Assembly programming]  In this question you will write a new DLX assembly language program and simulate it on the DLX simulator.  DLX is the name of a simple RISC ISA.  Before attempting to do this question, familiarize yourself with the DLX ISA and how to use the software by reading all the documents in the "DLX Software Details" link on Moodle.

Write a DLX assembly language programs that takes a single positive integer A, computes ceiling(log2 A), and prints out the answer on the screen.  For example, if A=13 or A=16 the answer should be 4; if A=20 or A=32 the answer should be 5.  In writing the program, do not attempt to call any library function for log; instead, repeatedly integer-divide the number by two, until the quotient is zero.  To integer-divide, do not use a division instruction; instead use the appropriate right-shift instruction.  Please write your entire code in the _main function itself.

Your program should not take any input from the screen.  Instead, it should hard-code in the value of A in the program.  Do this by declaring a global variable A in your program, and initializing it to its known constant value, such as 13.  (To see how to declare an initialized global, look at example program add.s/ in ~/dlx/test.)

Example: If A = 13, then your DLX session should look like the following:

> dlxsim
(dlxsim) load log2.s
(dlxsim) go _main
Log of input is 4
TRAP #0 received

When debugging your program, you may find it helpful to use DLXSIM to step through the program instruction by instruction, examine the values of registers during execution and set breakpoints.  These are very easy to do -- use the DLX simulator manual to learn to do these tasks and try them out for your program.

Please submit the following:

·       Program .s file (assembly program) you wrote.

·       Screenshot of a dlxsim session of your program running for A=13. 

·       Screenshot another dlxsim session of your program running for A=25, but this time, also use DLXSIM to tell you statistics on the program.  Use the "stats" command with the "all" option in DLXSIM to do so.  On the screenshot, highlight or circle the total number of cycles your program took to execute.

·       Combine all 3 files and upload to course on Moodle.
